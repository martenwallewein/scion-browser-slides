SCION Browser:
Possibilites and Pitfalls

Marten Gartner, Matthias Frei, David Hausheer

2021

Idea

  • Low entry bar for new users interested in SCION
  • Should provide access to (web)apps deployed on SCIONLab
  • Aim: 1-Click installation
  • So new users can get in touch with SCION without complex setup
  • Note: Should not replace scion-apps

Initial Concept

  • On local machine runs browser + SCION browser extension
  • In a VM (or on an external node) runs a proxy component
  • Looks easy, but...

Pitfalls

  • Determine which URLs are SCION URLs -> Regex for full URLS, but what about host names?
  • SCION address scheme not supported by browsers
  • Debugging HTTPS proying is "impossible" (quote from chrome bugtracker)

The HTTPS Problem

  • Main browsers restrict certain features to only secure contexts
  • This would require either to have the proxy handle incoming HTTPS or proxy the TLS session via HTTP Connect
  • Core Problem: We need certificates signed by valid CAs, which we don't have in SCIONLab, otherwise users have to accept self-signed certs

Electron: A potential Solution

  • We found a custom browser based on electron with a UI written in React
  • Solves the certificate problem: We can enable self-signed certificates in electron without user interference
  • Could handle SCION addresses
  • Furthermore, provides potential to adapt behaviour and UI to SCION

The proying itself...

  • Matthias Frei provided a basic prototype called skip that enables forward proxying over SCION
  • Version 2 handles HTTP Protocol mismatches (HTTP1 over TCP vs HTTP/2 over QUIC/SCION)
  • Handles HTTP Connect (for https proxying and websockets)

Whats Next?
